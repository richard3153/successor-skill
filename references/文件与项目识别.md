# 文件与项目识别指南

## 常见项目类型识别

### 代码项目

| 类型 | 识别文件 | 关键检查 |
|------|----------|----------|
| Python | `requirements.txt`, `pyproject.toml`, `setup.py` | 虚拟环境、依赖安装 |
| Node.js | `package.json`, `node_modules/` | npm install 是否完成 |
| Go | `go.mod`, `go.sum` | go mod download |
| Rust | `Cargo.toml`, `Cargo.lock` | cargo check |
| Java | `pom.xml`, `build.gradle` | 编译环境 |

### 数据项目

| 类型 | 识别文件 | 关键检查 |
|------|----------|----------|
| 数据分析 | `.csv`, `.xlsx`, `.sqlite` | 数据完整性 |
| 机器学习 | `model.*`, `.pkl`, `.h5` | 模型文件存在 |
| 报告生成 | `.docx`, `.pdf` | 输出文件状态 |

### 文档项目

| 类型 | 识别文件 | 关键检查 |
|------|----------|----------|
| 技术文档 | `README.md`, `docs/` | 目录结构 |
| 文档协作 | `.docx`, 金山文档链接 | 外部协作状态 |
| 知识库 | `memory/`, `references/` | 笔记完整性 |

## 项目状态评估

### 🟢 可继续（Green）

- 所有核心文件存在
- 依赖已安装（node_modules/, venv/ 等）
- 无编译错误
- README 中有清晰的运行说明

### 🟡 可继续但需补充（Yellow）

- 核心文件存在，但部分依赖缺失
- 配置文件不完整（如缺少 `.env`）
- Git 有未提交的更改

### 🔴 无法继续（Red）

- 核心文件缺失或损坏
- 依赖严重不兼容
- 项目结构被破坏

## 快速扫描命令

```powershell
# 列出所有文件（按修改时间排序，最近的在最前）
Get-ChildItem -Recurse -File | Sort-Object LastWriteTime -Descending | Select-Object FullName, LastWriteTime -First 50

# 查找所有 README 文件
Get-ChildItem -Recurse -Filter "README*" -File

# 查找所有配置文件
Get-ChildItem -Recurse -Include "package.json","requirements.txt","pyproject.toml","Cargo.toml","pom.xml" -File

# 检查 Git 状态
git status
git log --oneline -5

# 查找大型文件（可能需要清理）
Get-ChildItem -Recurse -File | Where-Object { $_.Length -gt 10MB } | Select-Object FullName, @{N='SizeMB';E={[math]::Round($_.Length/1MB,2)}}
```
