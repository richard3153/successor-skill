# 接替状态评估

## 评估维度

### 1. 对话历史完整性（权重：30%）

| 等级 | 描述 | 行动 |
|------|------|------|
| 高 | 上下文文件包含完整任务描述 | 直接继续 |
| 中 | 部分上下文，缺少细节 | 分析后继续 |
| 低 | 无上下文或上下文损坏 | 询问用户 |

### 2. 任务进展度（权重：25%）

评估当前任务的完成阶段：
- **0-25%**：概念阶段，刚开始
- **25-50%**：执行阶段，中期
- **50-75%**：收尾阶段，后期
- **75-100%**：接近完成

### 3. 记忆覆盖度（权重：20%）

MEMORY.md 中是否记录了：
- 当前项目的目标
- 关键决策历史
- 已解决的问题
- 已知风险

### 4. 项目文件状态（权重：15%）

- 工作区是否有未提交的变更
- 是否有待处理的临时文件
- 是否有 lock 文件或运行中的进程

### 5. 周期性任务（权重：10%）

检查是否有需要立即执行的：
- cron 定时任务
- heartbeat 待检查项
- 周期性提醒

## 综合评分

`
总分 = 历史完整性×0.3 + 进展度×0.25 + 记忆覆盖度×0.2 + 文件状态×0.15 + 周期性任务×0.1
`

| 总分 | 状态 | 建议行动 |
|------|------|----------|
| 80-100 | 🟢 优良 | 直接继续，快速交接 |
| 60-79 | 🟡 良好 | 补充分析后继续 |
| 40-59 | 🟠 一般 | 需要用户确认后继续 |
| 0-39 | 🔴 不足 | 询问用户，从头开始 |
