# 文件与项目识别

## 识别策略

### 按文件类型

| 类型 | 特征文件 | 说明 |
|------|----------|------|
| 代码项目 | package.json, Cargo.toml, go.mod | 确认包管理器和依赖 |
| Python | equirements.txt, pyproject.toml | 确认虚拟环境和依赖 |
| 文档项目 | docs/, README.md | 确认文档工具链 |
| 前端项目 | package.json, ite.config.js | 确认构建工具 |

### 按目录结构

`
.
├── src/           # 源代码
├── dist/          # 构建产物（勿提交）
├── tests/         # 测试文件
├── docs/          # 文档
├── scripts/       # 自动化脚本
├── data/          # 数据文件
├── temp/          # 临时文件（可清理）
├── output/        # 产物输出
├── logs/          # 日志文件
└── .env           # 环境变量（勿提交）
`

### 按 Git 状态

- Untracked files：新文件，需要判断是否应加入版本控制
- Modified files：已修改的文件，检查变更内容
- Staged files：已暂存，准备提交的文件
- Deleted files：已删除，需要确认是否有意为之

### 项目健康度检查

1. 运行 git status 查看工作区状态
2. 检查 .gitignore 是否合理
3. 确认没有敏感信息泄露（密码、密钥等）
4. 检查依赖是否完整（
pm install / pip install）
5. 确认构建是否能通过（
pm run build / cargo build）
