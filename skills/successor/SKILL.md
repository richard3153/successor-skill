# 继承者（Successor）— 智能体接替技能

## 核心职责

当原智能体达到沟通上限无法继续时，继承者自动承接以下职责：

1. **上下文迁移** — 读取上下文文件，获取对话历史、决策过程
2. **任务状态衔接** — 分析当前任务进展，识别未完成项
3. **记忆接管** — 读取 MEMORY.md 和 daily notes，继承长期记忆
4. **文件/项目接管** — 识别工作区文件，了解项目结构与状态
5. **断点续传** — 基于已有信息，无缝衔接工作，不遗漏任何关键上下文

## 使用场景

- 原智能体报告「达到沟通上限」「即将结束会话」
- 需要在新会话中延续之前的复杂任务
- 多人协作时，新智能体需要接手前任的工作

## 核心流程

### 步骤一：读取上下文文件

按以下优先级读取文件：

1. SOUL.md — 身份与行为规范
2. MEMORY.md — 长期记忆与重要决策
3. memory/YYYY-MM-DD.md — 最近几天的会话日志
4. AGENTS.md — 工作区说明
5. HEARTBEAT.md — 周期性任务提醒
6. TOOLS.md — 本地工具配置

### 步骤二：接替状态评估

对以下维度进行评估：

- **对话历史完整性**：上下文文件中是否有完整的任务描述
- **任务进展度**：当前任务处于哪个阶段，还有多少工作量
- **记忆覆盖度**：长期记忆中是否有关于当前任务的记录
- **项目文件状态**：工作区中是否有未提交的变更或待处理文件
- **周期性任务**：是否有需要立即执行的 cron/heartbeat 任务

### 步骤三：上下文文件索引

系统会按优先级展示所有上下文文件，供后续使用。

### 步骤四：执行任务延续

基于评估结果，确定最优的接替策略：
- 任务明确且有进展 → 直接继续执行
- 任务有记录但进展不明 → 先分析再继续
- 任务无记录 → 询问用户确认

## 断点续传规范

- **不假设**：不假设前任的工作状态，一切基于文件证据
- **不重复**：不重复已完成的工作
- **有记录**：将接替过程记录到 memory/YYYY-MM-DD.md
- **有确认**：重大决策前先向用户确认
